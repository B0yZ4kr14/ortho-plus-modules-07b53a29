Ortho+ Enterprise Architecture

Documentação Técnica Definitiva (v2.0) - Express & Prisma Stack

Visão Executiva

O Ortho+ é um sistema robusto refatorado a partir de sua estrutura Legacy (Edge Functions/Deno) para um Monolito Modular Moderno rodando nativamente no Node.js com Express e Prisma.

A separação em Domain-Driven Design e isolamento de domínios garante a integridade dos limites operacionais de cada módulo da clínica.

🧩 Padrões de Projeto (Frontend)

Camada React / Vite

  • Isolamento de React Hooks: Todo estado do lado do servidor é roteado por intermédio do useQuery do @tanstack/react-query, apontando estritamente para endpoints express /api/*.
  • Payload Enforcement: Nenhum DTO de formulário é disparado sem passar validamente pelos TypeGuards do Zod + react-hook-form.
  • Router Guards Inflexíveis: A verificação de ACL é tripla. O componente ProtectedRoute avalia a prop moduleKey injetada, comparando-a dinamicamente via AuthContext com a tabela clinic_modules persistida na Auth Store. 
src/modules/[nome-do-modulo]/
 ├── components/    # Componentes puros da UI (ex: Modais, Tabelas)
 ├── pages/         # Contêineres injetores de lógica
 ├── hooks/         # Orquestradores de mutations/queries
 └── utils/         # Helpers isolados do ecosistema global

⚙️ Topologia (Backend & Infraestrutura)

Express.js Gateway

A pipeline REST central é gerenciada pelo VM200, atuando como o motor assíncrono. Substitui o antigo acoplamento BaaS.

  • Sistema de Middleware: JWT Decoders robustos rejeitam fluxos sem assinatura RSA. Validam o clinic_id automaticamente e isolam sub-tenants por RLS.
  • Node Real-Time Engine (SSE): O socket `/api/events/stream` opera sob Server-Sent Events, despachando sub-eventos para listeners específicos sem sobrecarga TCP de WebSockets (Socket.io).
  • Cache Engine: Endpoints intensivos em leitura (e.g. Dashboard de BI, Analise Financeira) são cobertos por um buffer Redis com TTL inteligente, eliminando gargalos de banco. 
backend/src/
 ├── controllers/     # Handlers de Request delegando para Prisma
 ├── routes/          # Bindings Express (REST + Middleware bindings)
 ├── infrastructure/  # Singleton Wrappers para Logger, Queue, Redis
 ├── modules/         # Lógicas específicas de backend (Dumps, Exporting)
 └── utils/           # Type guards, Prisma Singletons

🗄️ Estrutura de Domínios Relacionais (Database)

Hospedado no VM201 de modo isolado da computação HTTP. O banco está sob governança estrita de permissões e triggers criptográficas.

  • Multi-Tenancy (Isolamento Row-Level): Todas as master schemas carregam clinic_id FK indexada nativamente. RLS Policies previnem select sem injeção de ID.
  • Continuous Audit: Cada mutation significativa gera um append na engine de logs acoplada pelo trigger after_transaction_audit no PostgreSQL.

🚀 Integração Contínua (CI/CD)

A engine de Deployment no GitHub garante zero regressão por cada merge na master.

  1. Serviços: Inicializa containers nativamente via services Action (PostgreSQL e Redis).
  2. Daemon Test: Inicializa a thread do Prisma npx prisma generate && npx tsx src/index.ts & e captura PID localmente.
  3. Playwright Test E2E: Shard em 2 cores rodando por completo suítes críticas com limit timeout total de 30 segundos usando Webkit, Firefox e Chromium.
