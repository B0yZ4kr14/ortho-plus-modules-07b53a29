## 🏗️ Arquitetura Implementada

### Domain-Driven Design (DDD)

Todos os módulos principais seguem a estrutura de 4 camadas:

```
src/modules/{module}/
├── domain/              # Regras de Negócio
│   ├── entities/       # Entidades (Campaign, Produto, Lead)
│   ├── valueObjects/   # Value Objects (Money, Email)
│   └── repositories/   # Interfaces (IRepository)
│
├── application/         # Casos de Uso
│   └── use-cases/      # CreateEntity, UpdateEntity, etc.
│
├── infrastructure/      # Implementações
│   ├── repositories/   # PostgreSQLRepository
│   └── mappers/        # Domain <-> DB
│
└── presentation/        # UI
    ├── hooks/          # useEntities
    ├── components/     # EntityList, EntityForm
    └── pages/          # EntityPage
```

### 3. Queries Otimizadas (N+1 Eliminado)

**Antes (N+1)**:
```typescript
// 1 query para tratamentos + N queries para prontuários
const tratamentos = await getTratamentos();
for (const t of tratamentos) {
  t.prontuario = await getProntuario(t.prontuarioId); // ❌ N queries
}
```

**Depois (JOIN)**:
```typescript
// 1 query única com JOIN
const tratamentos = await apiClient
  .from('pep_tratamentos')
  .select('*, prontuarios(*)')  // ✅ 1 query
  .eq('clinic_id', clinicId);
```
